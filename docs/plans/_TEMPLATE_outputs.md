## Phase-Specific Artifacts

### Inception Phase

Expected artifacts specific to Inception:

1. **Vision Document** - High-level project vision and scope
2. **Initial Use Case Model** - Key use cases identified (20-30% detailed)
3. **Glossary** - Domain terminology
4. **Initial Risk Assessment** - Major risks identified
5. **Business Case** - Why the project should proceed
6. **Initial Architecture Notebook** - Architectural approach
7. **Iteration Plan** - Plan for Elaboration phase
8. **Development Environment Setup** - Basic tooling and environment

### Elaboration Phase

Expected artifacts specific to Elaboration:

1. **Refined Use Case Model** - Use cases detailed (80-90% complete)
2. **Software Architecture Document** - Architectural decisions documented
3. **Design Model** - High-level design diagrams
4. **Executable Architecture Baseline** - Working skeleton system
5. **Risk List (Updated)** - Risks addressed or mitigated
6. **Test Strategy** - Overall testing approach
7. **Development Case** - Adapted development process
8. **Iteration Plan** - Plan for Construction phase

### Construction Phase

Expected artifacts specific to Construction:

1. **Complete Use Case Model** - All use cases implemented
2. **Working Software** - Production-ready features
3. **Test Suite** - Comprehensive automated tests
4. **User Documentation** - Guides and manuals
5. **Deployment Package** - Installable/deployable software
6. **Defect Reports** - Known issues and resolutions
7. **Updated Design Model** - As-built design documentation
8. **Iteration Plan** - Plan for Transition phase

### Transition Phase

Expected artifacts specific to Transition:

1. **Production Release** - Deployed software
2. **Release Notes** - What's new and what's changed
3. **Training Materials** - User and admin training
4. **Support Documentation** - Troubleshooting guides
5. **Post-Deployment Report** - Deployment results
6. **Lessons Learned** - Project retrospective
7. **Maintenance Plan** - Ongoing support approach
8. **Project Closure Report** - Final project summary

## Quality Standards

All artifacts must meet these standards:

### Documentation Standards

- Clear, concise language
- Proper grammar and spelling
- Consistent formatting (follow templates)
- Version controlled
- Reviewed and approved

### Code Standards

- Follows project conventions (see `docs/how-to-work/conventions.md`)
- Has automated tests
- Passes linting and type checking
- Code reviewed
- Documented (inline and external)

### Test Standards

- Covers all acceptance criteria
- Automated where possible
- Reproducible
- Maintains test coverage thresholds
- Includes edge cases
